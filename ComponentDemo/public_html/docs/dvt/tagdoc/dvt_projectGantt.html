<dvt:projectGantt>

dvt:projectGantt projectGantt project gantt

UIComponent class: oracle.adf.view.faces.bi.component.gantt.UIProjectGantt
Component type: oracle.dss.adf.gantt.ProjectGantt

The ProjectGantt is used to illustrate a list of tasks in a project schedule. It comprises of two major parts; the left side of the Project Gantt shows information about the tasks in the form of columns in the Table, while the right side of the Project Gantt displays graphical representation of the start and finish dates of tasks.

This component can have the following children: ganttLegend, collectionDragSource, collectionDropTarget, dragSource, dropTarget

The ProjectGantt provides predefined styles for the following task types out of the box:

  • Normal
  • Summary
  • Milestone

In addition, based on the data supplied by the Gantt model, the ProjectGantt can also display:

  • Progress information of a task (except for Milestone)
  • Actual start and end time of a task (except for Summary)
  • Whether the task is critical
  • Whether the task recurs over the period of time
  • Whether the task splits into two segments with an inactive time period between the segments

Applications can define their own styles or make changes to predefined styles by registering their own TaskbarFormat object through the TaskbarFormatManager. For details, please see the javadoc for oracle.adf.view.faces.bi.component.gantt.TaskbarFormat and oracle.adf.view.faces.bi.component.gantt.TaskbarFormatManager classes.

The Project Gantt supports direct manipulation of tasks on the chart. For example, users can change the start time or the duration of a task by dragging the task bar. Users can split a task into multiple segments, as well as establish dependency between tasks by simple mouse gestures.

Gantt Model

The ProjectGantt component uses a model to access the data in the underlying list. Like the TreeTable component, the ProjectGantt supports hierarchical data. The specific model class is org.apache.myfaces.trinidad.model.TreeModel.

The collection of objects returned by TreeModel must have, at the minimal, the following properties:

  • taskId - the id of the task
  • startTime - the start time of the task
  • endTime - the end time of the task

Optionally, the object could implement the oracle.adf.view.faces.bi.model.Task interface to ensure it provides the correct properties to the Gantt.

Columns

A user would specify what columns to display in the left side of the Gantt by adding the Column components as its children. For more information see the documentation for <af:column>.

Like the TreeTable, a "nodeStamp" facet is used to specify the primary identifier of an element in the hiearchy. For example, the "Task Name" Column might be used as the nodeStamp for the Project Gantt.

Legend

A legend can be attached to the Gantt by adding the ganttLegend component. The legend toggle button in the toolbar would only be available if a legend is attached and is visible.

Drag And Drop

The ProjectGantt can be made as a drop target for Table, TreeTable by using the dropTarget tag. Note that the Table, TreeTable must first be specified as a drag source before their content can be drop onto the Gantt. Please see the documentation for <af:dragSource>.

The ProjectGantt can be made as a drag source for Table, TreeTable by using the dragSource tag. Note that the Table, TreeTable must first be specified as a drop target before their content can be drop onto the Gantt. Please see the documentation for <af:dropTarget>.

Geometry Management

  • This component can be stretched by a parent layout component that stretches its children, e.g. panelStretchLayout, panelSplitter
  • When stretching this component, the data treeTable and chart treeTable will be stretch both horizontally and vertically. The toolbar and legend of the Gantt will only be stretched horizontally.
  • This component is automatically stretched to fill up the browser's viewport when it is the "sole visual root component" in the component tree and if the document component is configured with maximized="true" (the default).
  • When NOT stretched, this component will be displayed in its default dimensions which are provided by the skin. This can be modified by applying a dimension using the inlineStyle or styleClass attributes.

Examples

             
                <dvt:projectGantt id="gantt1"
                                  summary="gantt"
                                  startTime="2008-04-12" 
                                  endTime="2009-04-12" 
                                  value="#{project.model}" 
                                  var="task">
                    <f:facet name="major">
                        <dvt:timeAxis scale="months"/>
                    </f:facet>
                    <f:facet name="minor">
                        <dvt:timeAxis scale="weeks"/>
                    </f:facet>
                    <f:facet name="nodeStamp">
                        <af:column id="c1" headerText="Task Name">                        
                            <af:outputText id="ot1" value="#{task.taskName}"/>
                        </af:column>
                    </f:facet>
                    <af:column id="c2" headerText="Resource">
                        <af:outputText id="ot2" value="#{task.resourceName}"/>
                    </af:column>
                    <af:column id="c3" headerText="Start Date">
                        <af:outputText id="ot3" value="#{task.startTime}"/>
                    </af:column>
                    <af:column id="c4" headerText="End Date">
                        <af:outputText id="ot4" value="#{task.endTime}"/>
                    </af:column>
                </dvt:projectGantt>
